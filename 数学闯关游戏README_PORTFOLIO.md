# MathAdventure（数学大冒险）— AI 协作开发的小学数学闯关游戏

> **⚠️ 声明：本项目为个人独立完成的教育类 Web 游戏，无保密数据，源码可展示。项目全程使用 AI（Claude）辅助设计与开发，体现了"人机协作"的高效产出模式。**

---

## 一、项目背景

中国小学数学教育覆盖 6 个年级、12 个学期、近百个知识单元。市面上的数学练习 App 多为题库刷题模式，孩子容易枯燥放弃。

**核心设想：** 能否用 AI 把每一道数学题变成一个"闯关游戏"，让孩子在玩中练？

**结果：** 借助 AI 辅助设计与编码，从零到一完成了一个覆盖人教版小学 1-6 年级全部数学知识的闯关游戏平台，包含 **94 个独立游戏关卡**，每个关卡都是一个完整的交互式小游戏。

## 三、核心功能模块

### 3.1 通用游戏容器（Convention-over-Configuration）

**零构建工具的模块热加载系统：**

```
game.html?id=lvl_4_3_2
    ↓ 解析 URL 参数
levels.json → 查找 levelData
    ↓ 读取 gameType: "g4_d_u9_chicken_rabbit"
命名约定推导路径 → ./src/games/grade4/g4_d_u9_chicken_rabbit.js
    ↓ 动态 <script> 注入
window.CurrentGameModule.init('#game-root', levelData)
```

**设计决策：** 不用任何打包工具（Webpack/Vite），通过命名约定实现模块自动发现。新增一个关卡只需：

1. 写一个 JS 文件
2. 在 `levels.json` 加一条记录
3. 完成。零配置，零构建。

### 3.2 关卡设计模式（94 个模块统一规范）

每个关卡严格遵循三阶段渐进难度：

| 阶段             | 目标   | 题型示例                    |
| -------------- | ---- | ----------------------- |
| Phase 1 · 概念识别 | 建立直觉 | 点击 emoji 数数、拖拽比较符号、认读钟面 |
| Phase 2 · 核心计算 | 巩固技能 | 口算填空、竖式计算、方程求解          |
| Phase 3 · 综合应用 | 迁移运用 | 鸡兔同笼（3 种解法）、植树问题、逻辑推理   |

**程序化出题保证可重玩性：**

- 数值范围随机化（如鸡兔同笼：头数 `randInt(6,12)`，脚数 `randInt(16,34)`）
- 干扰选项算法生成（正确答案 ± `randInt(1,5)`）
- 每次游玩题目不同，避免死记硬背

### 3.3 交互方式矩阵

| 交互类型   | 技术实现                 | 适用场景          |
| ------ | -------------------- | ------------- |
| 点击选择   | DOM click 事件         | emoji 计数、图形辨认 |
| 拖拽操作   | HTML5 Drag & Drop    | 比较符号放置、凑十法    |
| 文本输入   | `<input>` + Enter 提交 | 口算、方程、应用题     |
| 多选网格   | Checkbox 网格          | 找因数、找倍数       |
| SVG 绘图 | SVG + mouse 事件       | 分数切披萨         |
| 时钟交互   | mouse/touch 旋转       | 认识钟表          |
| 线索排除   | 表格标记消除               | 逻辑推理题         |

### 3.4 游戏化系统

| 机制        | 实现                               |
| --------- | -------------------------------- |
| **星级评价**  | 0 错误 = ⭐⭐⭐，1-2 错 = ⭐⭐，3+ 错 = ⭐   |
| **金币奖励**  | 每关 10-50 币（按难度递增）                |
| **关卡解锁**  | 线性推进，完成当前关自动解锁下一关                |
| **知识掌握度** | 0-100 分值追踪，答对 +5，答错 -3           |
| **错题本**   | 自动记录每道错题（题目 / 错误答案 / 正确答案 / 时间戳） |
| **称号系统**  | "初出茅庐" → 更高成就（基于金币积累）            |

### 3.5 家长看板

`dashboard.html` 提供家长视角的学习分析：

- **统计卡片**：总金币、已完成关卡数、错题总数
- **知识雷达图**：Chart.js 绘制，各知识点掌握度一目了然
- **错题回顾**：展示错题详情，支持一键清空

## 七、工程实践

- **Convention-over-Configuration**：游戏模块通过命名约定自动发现，无需注册或配置
- **全局单例模式**：StateManager 作为唯一状态源，所有模块通过 `window.GameManager` 访问
- **共享工具库复用**：GameHelpers 封装了结算屏幕、选择题渲染、错误动画等通用组件，94 个模块零重复代码
- **CSS 去重注入**：`injectStyles(styleId, css)` 通过 ID 去重，防止模块切换时样式污染
- **渐进式代码风格演进**：一年级（对象字面量）→ 二年级起（IIFE + 独立函数），体现了规范的逐步成熟
- **程序化内容生成**：随机参数 + 算法干扰项，一个关卡可无限重玩
- **防御性编程**：localStorage 损坏自动恢复、错题本 100 条上限防止溢出
- **零依赖部署**：纯静态文件，双击 index.html 即可运行（需 HTTP server 加载 JSON）
